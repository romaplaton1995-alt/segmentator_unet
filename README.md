# 3D Brain Tumor Segmentation

Программный комплекс для многоклассовой сегментации опухолей головного мозга по четырём модальностям МРТ и интерактивной визуализации результата.

Проект предназначен для учебных и исследовательских целей. Он не является медицинским изделием, не заменяет заключение врача и не должен использоваться для самостоятельной постановки диагноза или выбора лечения.

## 1. Возможности

- загрузка четырёх NIfTI-модальностей:
  - FLAIR;
  - T1;
  - T1ce;
  - T2;
- проверка входных файлов;
- предобработка медицинских изображений;
- применение трёхмерной модели 3D U-Net;
- многоклассовая сегментация;
- построение 3D-модели;
- экспорт результата в формат GLB;
- интерактивная визуализация через Three.js;
- просмотр сагиттального, фронтального и аксиального срезов;
- управление видимостью слоёв;
- управление масштабом и прозрачностью;
- запуск через Docker Compose.

## 2. Классы сегментации

| Код | Класс |
|---:|---|
| 0 | Фон |
| 1 | Некроз |
| 2 | Отёк |
| 3 | Активная опухоль |

Коды классов должны совпадать с кодами, использованными при обучении модели. При изменении схемы разметки необходимо одновременно изменить модель, обработку масок, вычисление метрик и визуализацию.

## 5. Системные требования

Для запуска необходимы:

- Docker Engine;
- Docker Compose;
- свободные порты 3000 и 8000;
- видеокарта NVIDIA и NVIDIA Container Toolkit для GPU-инференса;
- не менее 6 ГБ видеопамяти для конфигурации, использованной при разработке;
- свободное место для Docker-образов и временных файлов;
- четыре согласованных NIfTI-файла одного исследования.

Запуск на CPU может быть возможен, однако время обработки будет больше.

## 6. Установка

Клонируйте репозиторий:

```bash
git clone https://github.com/romaplaton1995-alt/segmentator_unet.git
```

Перейдите в каталог проекта:

```bash
cd segmentator_unet
```

Проверьте наличие файла весов:

```text
weights/best_model.pth
```

Если файл имеет другое имя, измените путь к checkpoint в backend или переименуйте файл.

## 7. Запуск

Для сборки и запуска проекта выполните:

```bash
docker compose up --build
```

Для запуска в фоновом режиме:

```bash
docker compose up --build -d
```

Для остановки контейнеров:

```bash
docker compose down
```

После запуска откройте в браузере:

```text
http://localhost:3000
```

Адрес проверки backend:

```text
http://localhost:8000/health
```

Документация API, если она включена в конфигурацию:

```text
http://localhost:8000/docs
```

## 8. Использование интерфейса

1. Откройте `http://localhost:3000`.
2. Выберите файл FLAIR.
3. Выберите файл T1.
4. Выберите файл T1ce.
5. Выберите файл T2.
6. Убедитесь, что все файлы относятся к одному исследованию.
7. Нажмите «Начать анализ».
8. Дождитесь завершения обработки.
9. Просмотрите 3D-модель и ортогональные срезы.
10. Используйте переключатели для управления отображением слоёв.
11. Для удаления результата нажмите «Очистить результат».

## 9. Входные данные

Поддерживаются файлы:

```text
.nii
.nii.gz
```

Необходимо загрузить четыре модальности:

```text
FLAIR
T1
T1ce
T2
```

Файлы должны иметь:

- одинаковую размерность;
- совместимое пространственное разрешение;
- одинаковую ориентацию;
- принадлежность к одному исследованию.

Проект не исправляет произвольное несоответствие геометрии и не выполняет полноценную клиническую регистрацию изображений, если такая функция не реализована в текущей версии кода.

## 12. Возможные ошибки

### Модель не найдена

Проверьте наличие файла:

```text
weights/best_model.pth
```

и путь к нему в `backend/inference.py`.

### Несовместимый checkpoint

Убедитесь, что архитектура в `model_unet.py` совпадает с архитектурой, использованной при обучении модели.

### Несовместимые размеры

Проверьте размеры и spacing всех четырёх NIfTI-файлов. Они должны соответствовать одному исследованию.

### Backend не отвечает

Проверьте состояние контейнеров:

```bash
docker compose ps
```

Просмотрите журнал backend:

```bash
docker compose logs backend
```

Проверьте endpoint:

```text
http://localhost:8000/health
```

### Frontend не загружается

Просмотрите журнал frontend:

```bash
docker compose logs frontend
```

Затем пересоберите контейнер:

```bash
docker compose build --no-cache frontend
docker compose up
```

### GLB не построен

Возможные причины:

- маска сегментации пустая;
- результат содержит слишком маленькие компоненты;
- произошла ошибка построения поверхности;
- недостаточно оперативной или видеопамяти.

- проект не является медицинским изделием;
- результаты не заменяют работу врача;
- отсутствует независимая клиническая валидация;
- качество зависит от соответствия входных данных данным обучения;
- 3D-инференс требует значительных вычислительных ресурсов;
- небольшие или нетипичные области могут сегментироваться менее точно;
